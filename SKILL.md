# Contract Kit — 影像製作合約包產生器與審計工具

從模板產生完整合約文件包，並提供邏輯審計、一致性核對、文件完整性檢查。

## Mode 1: Interactive Menu（選單模式）

使用者啟動 skill 時，顯示以下選單：

```
=== Contract Kit ===

請選擇操作：

  [A] 新案建立 — 輸入基本資料，產生整包文件
  [B] 單檔產生 — 只產生特定文件（1-6）
  [C] 合約審計 — 對已產生的文件包做全套檢查
  [D] 補漏檢查 — 檢查案件文件完整性，提醒缺漏

輸入選項 (A/B/C/D):
```

### [A] 新案建立流程

**Step 1 — 收集基本資料**

向使用者詢問以下必填欄位（分組問，不要一次列 30 個）：

**第一組：甲乙方資訊**
- 甲方公司名稱、負責人/代表人、地址、統編、承辦人、電話
- 乙方公司/工作室名稱、負責人、地址、統編、承辦人、電話

**第二組：專案基本**
- 專案名稱
- 專案類型：影片 / 平面 / 混合
- 拍攝日期
- 交付物規格（解析度、格式、數量）

**第三組：財務**
- 總金額（含稅）
- 付款拆分比例（預設 50/50）
- 乙方匯款帳戶資訊

**第四組：時程節點**
- 簽約日、訂金日、拍攝日
- A copy / B copy / Final 各預定日期
- 結案日

**第五組：製作團隊**
- A-cam / B-cam 人員
- 設備清單（機身、鏡頭、燈光、收音）
- 集合時間、地點

每組收集完畢後確認，再進下一組。允許使用者跳過選填欄位（以 `—` 標記）。

**Step 2 — 產生文件包**

根據專案類型選擇合約模板（影片 → `contract-video.md`，平面 → `contract-photo.md`），將所有 `{{placeholder}}` 替換為實際值。

產生順序：
1. 合約書（主文件）
2. 製作時程表
3. 設備清單
4. 拍攝通告單
5. 鏡頭配置與分鏡表

每個文件產生後立即確認，使用者可即時修改。

**Step 3 — 自動執行 Mode C（合約審計）**

全包產生完畢後，自動跑一次審計，報告結果。

### [C] 合約審計（三層檢查）

需要使用者指定案件名稱（如「恬馨幼兒園」），然後讀取該案所有文件。

#### Layer 1: 合約邏輯審計

針對合約書本體：

| 檢查項 | 規則 |
|--------|------|
| 付款加總 | 首付 + 尾款 = 總金額，比例加總 = 100% |
| 日期邏輯 | 簽約日 ≤ 訂金日 ≤ 拍攝日 ≤ A copy ≤ B copy ≤ Final ≤ 結案日 |
| 驗收時限 | 驗收小時數 > 0、修改申請天數 > 0 |
| 解約通知 | 終止通知天數已填寫且合理（≥ 7 天） |
| 肖像授權 | 年限已填、續約費用已填（如有此條） |
| 著作權歸屬 | 著作人格權 vs 著作財產權歸屬明確 |
| 附件引用 | 合約書引用的附件一（時程）、附件二（報價）存在且非空 |
| 報價單加總 | 小計 + 稅金(5%) = 總計，與合約第五條金額一致 |
| 不可抗力 | 延遲費用分攤比例已約定 |
| 修改次數 | 是否明定免費修改次數或條件 |

#### Layer 2: 基本資料一致性核對

跨文件比對以下欄位，任何不一致標記為 `MISMATCH`：

| 欄位 | 應出現文件 |
|------|-----------|
| 甲方名稱 | 合約、時程、通告單 |
| 乙方名稱 | 合約、時程 |
| 專案名稱 | 全部文件 |
| 拍攝日期 | 合約、時程、通告單、設備清單、分鏡表 |
| 總金額 | 合約、報價單 |
| 訂金金額 | 合約、時程 |
| 尾款金額 | 合約、時程 |
| A-cam 人員 | 通告單、分鏡表 |
| B-cam 人員 | 通告單、分鏡表 |
| 設備/鏡頭 | 設備清單、分鏡表、通告單 |

**輸出格式**：

```
=== Cross-Reference Report: {{案件名}} ===

[PASS] 甲方名稱 — 全文件一致：「私立恬馨幼兒園」
[PASS] 專案名稱 — 全文件一致：「恬馨幼兒園形象影片」
[MISMATCH] 拍攝日期 — 合約: 4/10、通告單: 4/11  ← 需修正
[MISSING] 尾款金額 — 時程表中未標示

共 {{N}} 項 | PASS: {{n}} | MISMATCH: {{n}} | MISSING: {{n}}
```

#### Layer 3: 文件包完整性 + 時間線對齊

**文件完整性**：

根據專案類型，檢查必要文件是否都已產生：

影片專案必要文件：
- [ ] 合約書（影片版）
- [ ] 製作時程表
- [ ] 設備清單
- [ ] 拍攝通告單
- [ ] 鏡頭配置與分鏡表

平面專案必要文件：
- [ ] 合約書（平面版）
- [ ] 製作時程表

**時間線對齊**：

從所有文件中萃取日期，建立統一時間軸：

```
=== Timeline Alignment: {{案件名}} ===

04/01 簽約日          [合約 ✓] [時程 ✓]
04/01 收取訂金        [合約 ✓] [時程 ✓]
04/10 拍攝日          [合約 ✓] [時程 ✓] [通告單 ✓] [設備 ✓]
04/14 A copy          [合約 ✓] [時程 ✓]
04/15 A copy 修改清單 [時程 ✓]
04/17 B copy          [合約 ✓] [時程 ✓]
...

⚠ 通告單缺少集合時間
⚠ 設備清單日期與合約拍攝日不符
```

### [D] 補漏檢查

快速掃描指定案件，報告：

1. **文件缺漏** — 哪些必要文件尚未產生
2. **Placeholder 殘留** — 掃描所有文件中的 `{{...}}` 未填欄位
3. **空白表格** — 表格行全為空或只有模板預設值
4. **PDF 缺漏** — `.md` 已產生但 `pdf/` 中缺少對應 PDF

**輸出格式**：

```
=== Gap Report: {{案件名}} ===

📄 文件狀態
  ✅ 合約書 (.md ✓ / .pdf ✓)
  ✅ 時程表 (.md ✓ / .pdf ✓)
  ⚠️ 通告單 (.md ✓ / .pdf ✗) ← 需產出 PDF
  ❌ 鏡頭分鏡表 — 尚未建立

📝 Placeholder 殘留 (共 {{N}} 處)
  合約書: 無殘留
  時程表: {{日期}} ×2 處（第 3, 5 行）

📋 待辦
  1. 產生鏡頭分鏡表 → 選 [B] 再選 6
  2. 補填時程表日期 → 直接編輯 .md
  3. 產出通告單 PDF → /pdf 或手動轉換
```

## Cross-File Dependency Map

文件間的資料流向（上游變更 → 下游需同步）：

```
合約書 (master)
 ├─→ 製作時程表（日期、金額、甲乙方）
 ├─→ 報價單（嵌入合約附件二）
 │
 拍攝日確定後 ──→
 ├─→ 設備清單（日期、專案名）
 ├─→ 拍攝通告單（日期、地點、人員、rundown）
 └─→ 鏡頭分鏡表（人員、鏡頭、場景）
      │
      └─ 設備清單 ←→ 鏡頭分鏡表（鏡頭互相引用）
         通告單 ←→ 鏡頭分鏡表（人員、場景互相引用）
```

修改合約書的金額或日期時，提醒使用者同步更新下游文件。

---

## Audit Output Standards

- 每項檢查結果用 `[PASS]` / `[MISMATCH]` / `[MISSING]` / `[WARN]` 前綴
- 結尾附統計摘要
- 發現問題時給出具體修正建議（指出哪個文件的哪一行）
- 嚴重問題（金額不符、日期矛盾）用 `[CRITICAL]` 標記

---

## Guard Rails

- **不自動修改合約條款** — 發現問題只報告，不自動改（法律文件需人工確認）
- **金額計算** — 永遠用使用者提供的數字，不自行推算稅金除非使用者要求
- **個資處理** — placeholder 填入的個資（統編、電話、地址）不進 git commit message
- **模板更新** — 修改模板後提醒：「已有的實例不會自動更新，需手動同步或重新產生」
